# 任务：修复模拟结果不合理 - 倒数学生显示100%录取中上学校

## 背景

中考志愿模拟系统返回了不合理的结果：

**测试用例：**
- 学生排名：129/130（全班倒数第二）
- 总分：669.5
- 区县：嘉定区（districtId: 11）
- 初中：middleSchoolId: 48
- 有名额分配到校资格

**错误结果：**
```json
{
  "batch": "UNIFIED_1_15",
  "schoolId": 187,
  "schoolName": "上海市嘉定区嘉一实验高级中学",
  "probability": 100,
  "riskLevel": "safe",
  "volunteerIndex": 8
}
```

学校 187（嘉一实验高级中学）是嘉定区中上等级学校，不可能以 100% 概率录取全区倒数排名的学生。

一定要理解上海中考的录取规则。分批次录取，每个批次都遵守平行志愿的规则：
- 名额到区，区内排名竞争
- 名额到校，校内排名竞争
- 统一招生（1-15），区内排名竞争

平行志愿的基本规则是：
分数排序（也就是排名）：所有考生按中考总分从高到低排队。
依次投档：从第一名考生开始，依次处理其志愿。
志愿检索：轮到某考生时，系统从该考生的第一志愿开始逐一检索。
录取：当检索到某个志愿学校还有剩余名额时，即被投档录取，后续志愿自动作废。
滑档：如果所有志愿学校都已录满，则该考生本批次滑档，进入下一批次。

关键要点：
分数是王道（也就是排名）：高分考生永远比低分考生优先被检索，无论其志愿顺序。
志愿顺序很重要：对你自己而言，系统严格按你填报的顺序检索，把最想去的学校放在前面。
一档一投：每人每批次最多只能被投档录取一次。

**模拟过程支持三个批次。每个批次都是独立进行的（竞争对手的数量和排名不同）。要分别进行三次录取过程。当考生在任何一个批次被录取，后面的批次就不用执行了。**
整个过程像是多轮次的晋级赛，每个批次被录取的竞争对手都不会参加后面的批次的录取。越往后面，竞争对手越少。
所以每个批次的竞争对手人数不同。在开始模拟时，先一次性的生成区内的全部竞争对手，根据考生的校内排名，将这次竞争对手中的一部分人选为考生的校内竞争对手。
然后再开始分批次的录取。被录取的竞争对手要从后面的批次中退出。
