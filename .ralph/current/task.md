# 任务：前端多端适配重构

## 背景

当前上海中考招生模拟系统前端仅支持 Web 端（PC 浏览器）。为了扩大用户覆盖范围，需要重构前端架构，使其同时支持：

1. **Web** - PC 浏览器访问（现有功能）
2. **H5** - 移动端浏览器访问（响应式适配）
3. **微信小程序** - 微信生态内使用

## 技术选型

**采用 uni-app 框架**（基于 Vue 3）：

| 特性 | 说明 |
|------|------|
| 框架 | uni-app (Vue 3 + TypeScript + Vite) |
| UI 库 | uni-ui + 自适应组件 |
| 状态管理 | Pinia（保持现有） |
| 网络层 | Connect-RPC（保持现有） |
| 样式 | SCSS + 响应式设计 |

**选型理由**：
- 与现有 Vue 3 技术栈一致，学习成本低
- 一套代码编译到多端，维护效率高
- 社区成熟，文档完善
- 支持条件编译，平台差异化处理

## 成功标准

1. ✅ Web 端功能与现有版本一致
2. ✅ H5 端在移动浏览器正常使用
3. ✅ 微信小程序可提交审核
4. ✅ 三端代码统一维护，共享核心逻辑
5. ✅ 构建产物可独立部署

## 约束条件

1. **API 兼容**：后端 Connect-RPC 接口保持不变
2. **数据模型**：Protobuf 生成的类型定义复用
3. **功能等价**：核心功能（志愿填报、分析、历史记录）三端一致
4. **品牌统一**：三端视觉风格保持一致
5. **渐进迁移**：可分阶段发布，先 Web + H5，后小程序

```
frontend/
├── src/
│   ├── pages/              # 页面（uni-app 规范）
│   │   ├── index/          # 首页
│   │   ├── recommendation/ # 志愿推荐
│   │   ├── form/           # 志愿填报

│   │   └── history/        # 历史记录

│   │   └── business/       # 业务组件
│   ├── composables/        # 组合式函数（复用现有）
│   ├── stores/             # Pinia 状态（复用现有）
│   ├── api/                # API 层（复用 Connect-RPC）
│   ├── utils/              # 工具函数
│   ├── static/             # 静态资源
│   ├── uni_modules/        # uni-app 插件
│   ├── App.vue             # 应用入口
│   ├── main.ts             # 主入口
│   ├── manifest.json       # uni-app 配置
│   └── pages.json          # 页面路由配置
├── dist/
│   ├── build/h5/           # H5 构建产物
│   ├── build/web/          # Web 构建产物
│   └── dev/mp-weixin/      # 微信小程序开发产物

```

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Element Plus 不兼容小程序 | 高 | 使用 uni-ui 替代，封装统一组件 |
| Connect-RPC 小程序适配 | 中 | 封装适配层，处理网络请求差异 |
| 现有组件迁移工作量大 | 中 | 优先迁移核心功能，渐进式迁移 |
| 微信小程序审核要求 | 低 | 遵循小程序设计规范，提前测试 |

## 里程碑

- **Phase 1**: 项目初始化 + 基础架构
- **Phase 2**: 核心页面迁移
- **Phase 3**: 业务逻辑迁移
- **Phase 4**: H5 适配优化
- **Phase 5**: 微信小程序适配
- **Phase 6**: 测试与发布
